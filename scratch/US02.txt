US02 - Là người dùng, tôi muốn khai báo 1 code phí với các thông tin chi tiết về mức phí cho từng Phân khúc Khách hàng (KHCN, KHTC,...)  và điều kiện áp dụng để đảm bảo tính chính xác trong thanh toán.
Yêu cầu nghiệp vụ
Navigation: Khi muốn khai báo các code phí, người dùng truy cập chức năng: Tham số >> Danh mục sản phẩm dịch vụ >> SPDV cấp cuối >> Thêm mới Code phí >> Thiết lập Quy tắc tính phí (Quy tắc tính phí)

Code phí là cấp khai báo cuối cùng trong cây Danh mục SPDV, một code phí được gán trực tiếp với một SPDV cấp cuối. Các thông tin được khai báo bao gồm: Thông tin chung của Code phí, Điều kiện tính phí, Quy tắc tính phí
	•	Các thông tin chung của Code phí cần quản lý: Mã phí, Tên phí, Loại tiền tệ, VAT, Phí đã bao gồm VAT, Đối tượng thu phí, Loại Khách hàng, Loại tính phí, Tần suất, Ngày thu, Loại tiền tối thiểu/tối đa, Mã hạch toán category, Trạng thái
	•	Mã phí do hệ thống tự sinh và người dùng không được phép chỉnh sửa. Nguyên tắc tự sinh mã phí được mô tả tại US04 về Tự động sinh mã Code phí.
	•	Loại tiền tệ chính là Loại tiền tệ của Code phí.
	•	VAT: Giá trị % thuế suất VAT
	•	Trường hợp Code phí không phải chịu VAT, người dùng bỏ trống không nhập thông tin
	•	Trường hợp Code phí chịu VAT, bắt buộc nhập giá trị là số > 0 và <100 (Ví dụ thuế suất là 10% nhập số 10)
	•	Phí đã bao gồm VAT: Số tiền phí hệ thống tính ra đã bao gồm VAT hay chưa?
	•	Nếu ko tích chọn trường Phí đã bao gồm VAT, trường VAT nhập giá trị, số tiền phí thực thu của khách hàng = Số tiền phí hệ thống tính ra + VAT*Số tiền phí hệ thống tính ra
	•	Nếu tích chọn trường phí đã bao gồm VAT, trường VAT bắt buộc phải nhập giá trị là số >0 và <=100, số tiền phí thực thu của khách hàng = Số tiền phí hệ thống tính ra
	•	Đối tượng thu phí: Người dùng lựa chọn Khách hàng hoặc Merchant
	•	Khách hàng
	•	Merchant: Các đối tác của PVCB là đơn vị chấp nhận thanh toán
	•	Loại khách hàng: Bao gồm 4 loại khách hàng:
	•	KHCN: Khách hàng cá nhân
	•	KHTC: Khách hàng doanh nghiệp lớn/Khách hàng doanh nghiệp vừa và nhỏ
	•	DNSN: Khách hàng doanh nghiệp siêu nhỏ và hộ kinh doanh
	•	CBNV: Khách hàng là cán bộ nhân viên của ngân hàng
	•	Loại tính phí: Bao gồm 2 loại:
	•	Theo giao dịch
	•	Định kỳ
	•	Loại tiền tối thiểu/tối đa là Loại tiền tệ của Số tiền phí tối thiểu, Số tiền phí tối đa và có thể khác với Loại tiền tệ của Code phí
	•	Trạng thái của code phí bao gồm
	•	Chưa gán: Code phí chưa được gán với Biểu phí nào
	•	Hoạt động: Code phí đang gán với một Biểu phí Đang hiệu lực/Chưa hiệu lực
	•	Ngừng hoạt động: Các biểu phí mà Code trực thuộc đều đang ở trạng thái Hết hiệu lực 
	•	Hủy: Code phí khai báo sai thông tin và không thể tái sử dụng
Trạng thái mặc định của Code phí khi Thêm mới = Chờ gán
	•	Các loại điều kiện tính phí bao gồm: Điều kiện theo khách hàng, Điều kiện theo Tài khoản, Điều kiện theo Thẻ, Điều kiện theo giao dịch. 
	•	Trường điều kiện: Người dùng được chọn từ danh sách điều kiện có sẵn trong hệ thống và đã được cấu hình nguồn dữ liệu tương ứng.
	•	Operator là các toán tử được dùng để so sánh giữa Trường điều kiện và Giá trị
	•	Giá trị là trường người dùng nhập các giá trị cụ thể tương ứng với Trường điều kiện và làm cơ sở để so sánh khi thực hiện giao dịch
	•	Về Quy tắc tính phí: Người dùng được thiết lập công thức tính phí cho Code phí theo mô tả tại US06
Khi khai báo code phí, người dùng bắt buộc phải khai báo trước các thông tin về Loại tiền tệ, Loại khách hàng, Loại tính phí và Loại tiền tối thiểu/tối đa để làm căn cứ xác định các thông tin về Điều kiện tính phí và Quy tắc tính phí
	•	Loại tiền tệ: Người dùng chọn loại tiền tệ của code phí trước khi chọn loại tiền tệ tối thiểu/tối đa, trường loại tiền tối thiểu/tối đa sẽ tự động cập nhật theo loại tiền tệ của code phí
	•	Loại khách hàng
	•	Loại tính phí:
	•	Nếu Loại tính phí = Theo giao dịch, người dùng được phép khai báo Điều kiện tính phí theo Khách hàng, theo Giao dịch và theo Tài khoản
	•	Nếu Loại tính phí = Định kỳ, người dùng được phép khai báo Điều kiện tính phí theo Khách hàng và theo Tài khoản/Thẻ
	•	Loại tiền tối thiểu/tối đa: 
	•	Trường hợp người dùng chọn loại tiền tối thiểu/tối đa trước, chọn loại tiền tệ của code phí sau, hệ thống sẽ không cập nhật loại tiền tối thiểu/tối đa theo loại tiền của code phí
	•	Loại tiền tối thiểu/tối đa = VND/JPY, người dùng chỉ được nhập giá trị phí tối thiểu và tối đa dưới dạng số nguyên dương
	•	Loại tiền tối thiểu/tối đa <> VND/JPY, người dùng được phép nhập giá trị phí tối thiểu và tối đa dưới dạng số thập phân với tối đa 2 số sau dấu thập phân
	•	Sau khi Maker khai báo thành công, bản ghi Code phí sẽ ở trạng thái chờ duyệt với hành động là Thêm mới và chỉ hiển thị trên lưới Danh mục SPDV sau khi cấp duyệt cuối cùng phê duyệt theo nguyên tắc mô tả tại US25 về Ma trận phê duyệt.
Bản ghi Code phí sau khi duyệt thành công sẽ được hiển thị trên lưới chức năng và cho phép chỉnh sửa. Khi người dùng chỉnh sửa Code phí, hệ thống chỉ cho phép chỉnh sửa Trạng thái Code phí
	•	Chỉ cho phép chỉnh sửa thành trạng thái Hủy
	•	Sau khi Maker chỉnh sửa thành công, bản ghi Code phí sẽ ở trạng thái chờ duyệt với hành động là Sửa và chỉ hiển thị thông tin sau sửa trên lưới Danh mục SPDV sau khi cấp duyệt cuối cùng phê duyệt theo nguyên tắc mô tả tại US25 về Ma trận phê duyệt.
Hằng ngày, tại thời điểm đầu ngày, hệ thống tự động kiểm tra ngày hiệu lực và ngày hết hiệu lực của Biểu phí mà code phí trực thuộc, trạng thái hoạt động của code phí sẽ được cập nhật theo trạng thái hoạt động của biểu phí. Nguyên tắc cập nhật trạng thái được mô tả tại US06 về Khai báo Biểu phí.
