# PROMPT: ĐỒNG BỘ TEST PLAN TỪ TÀI LIỆU URD (v0.9+)

## Tình huống (Context)
Bạn là một Senior QA Lead. Bạn có một file Test Plan dạng Excel (`.xlsx`) và một tài liệu Yêu cầu nghiệp vụ (`.docx`). Khi URD có phiên bản mới, bạn cần kiểm tra xem Test Plan hiện tại đã đầy đủ các chức năng mới chưa và sửa các lỗi lệch số (mã chức năng).

## Nhiệm vụ (Objective)
1. **Kiểm tra và So sánh:** Đối chiếu danh mục tính năng (Feature List) trong URD với Test Plan.
2. **Phát hiện sai lệch:** 
   - Feature mới hoàn toàn (nhập mới).
   - Module bị đổi mã (ví dụ: SA.05 cũ đổi thành SA.06 mới).
   - Chi tiết danh sách (API, Phụ lục) chưa được bóc tách.
3. **Cập nhật (Auto-fix):** Tự động chèn dòng và sửa lại nhãn (labels) cho file Test Plan.

## Hướng dẫn thực hiện (Instructions)
- Sử dụng thư viện `python-docx` để đọc URD, tìm các Heading và Table (Danh sách API thường nằm trong bảng).
- Sử dụng `pandas` để đọc Sheet "Test Plan".
- Logic so sánh dựa trên "Mã chức năng" (PR.01, SA.01...) và "Tên chức năng".
- Khi chèn một Module mới (ví dụ SA.05), phải tịnh tiến (increment) toàn bộ mã số của các Module sau đó (SA.06, SA.07...).
- Sau khi cập nhật, phải tạo file mới: `{Original_Name}_Updated.xlsx`.
